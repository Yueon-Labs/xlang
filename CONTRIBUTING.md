# Contributing to NTS

NTS is currently in the language-design phase. Contributions are most useful when they improve clarity and reduce ambiguity.

## Good first contributions

- Improve examples in `examples/`.
- Tighten wording in `docs/spec-v0.1.md`.
- Add invalid examples with expected diagnostics.
- Propose simpler syntax when there are multiple possible forms.

## Design rule

If a feature makes AI-generated code less stable, less explicit, or harder to type-check, it should not be added to v0.1.
